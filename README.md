# changenow-2-0

ChangeNOW homepage on Next.js 16 (App Router, Turbopack) + React 19 + TypeScript.

## What's in here

- **Homepage** ŌĆö the hi-fi prototype (`ChangeNOW Homepage.html`) ported 1:1 into React, broken into 13 sections (`components/homepage/*`).
- **Footer** ŌĆö `@changenow/cn-ui-kit` Footer, re-skinned to the prototype's design tokens (see `components/layout/footer-overrides.css`).
- **i18n** ŌĆö same approach as `changenow-blog`: build-time fetch from Strapi ŌåÆ 16 per-locale JSON dicts ŌåÆ server `loadDict` (cached for a week) + client `LocalizationProvider`.
- **Caching** ŌĆö also same as the blog: `unstable_cache` for non-`fetch` work, route-segment `revalidate` for pages, `revalidateTag` warm-cache route, `experimental.staleTimes` for client navigations.
- **Routing** ŌĆö `proxy.ts` (Next 16 replacement for `middleware.ts`) rewrites non-prefixed paths to the default locale and redirects `/en/...` ŌåÆ `/...`.

## Calculator (`SwapWidget`) architecture

The hero's calculator is the most complex client surface in the app. It
covers four modes ŌĆö **Swap**, **Buy / Sell**, **Convert** (Pro spot/limit),
and **Private transfer** ŌĆö and creates real transactions on the Private
flow without bouncing to the legacy SPA.

### Layer cake


Each mode is **self-contained**. The orchestrator picks the active
`<Mode>View` based on `mode` and pipes state down via props. Mode
unmount on switch is wired through React's conditional render so the
old mode's effects, timers, and listeners drop cleanly.

### URL state sync ŌĆö `useHashSync`

A 60-line hook in `calculator/shared/useHashSync.ts` owns the two-way
sync between component state and `window.location.hash` + a `?mode=1`
query flag.

- On mount and on every `hashchange`, calls the caller's `read` with the
  parsed params; the caller dispatches setters per its mode mix.
- Whenever the dependency list changes, `write` is re-run. When it
  returns `null` (state matches initial defaults), the hook strips
  *both* the hash and the `?mode=1` flag ŌĆö fresh visits to `/` stay at
  `/`. When it returns `URLSearchParams`, the hash is set to their
  string form and `?mode=1` is added (other query params preserved).
- The `?mode=1` flag is the "user has interacted" marker. The server can
  read it during SSR to decide whether to render the Suspense skeleton or
  ship the calc with defaults pre-populated.

Per-mode hash adapters (`swap.url.ts`, `convert.url.ts`, ŌĆ”) carry only
the mode-specific keys (`rate`, `sub`, `fiatDir`, `provider`, `address`,
`extraId`). Common params (`from`, `to`, `amount`, `dir`, networks) live
in the orchestrator. The active mode's `parseHash` runs on every read;
`writeHash` runs only for the currently active mode.

### Estimate fetching ŌĆö `useExchangeEstimate`

A debounced (300 ms) + interval-refreshed (120 s) hook over
`/api/estimate`. Response shape:

```ts
interface EstimateResponse {
  fromAmount: number;
  toAmount: number | null;
  flow: 'standard' | 'fixed-rate';
  type: 'direct' | 'reverse';
  rateId: string | null;        // populated when flow=fixed-rate
  validUntil: string | null;    // ~5 min ahead when fixed-rate
  minAmount, maxAmount, isAmountInRange,
  withdrawalFee, depositFee,
  cashbackUsd,                  // NOW token ├Ś NOWŌåÆUSD price
  providers: EstimateProvider[],// fiat on-ramps for buysell
  // ...
}
```

The 2-minute refresh aligns with the upstream's ~5-minute `validUntil`
envelope so the `rateId` cached in state is always inside the validity
window when the user clicks Send. The hook auto-passes `useRateId=true`
to the API when `flow === 'fixed-rate'` ŌĆö the upstream issues a `rateId`
only on that branch, and the absence of a rateId blocks fixed-rate
transaction creation.

`allowSameAsset: true` opts the hook out of the cross-asset error guard.
This is what unlocks the Private transfer fee quote: same ticker on the
same network is rejected by the upstream's `/v1.3/exchange/estimate`
unless `source=private-transfers` is forwarded (the API route does this
when it sees `from === to && fromNetwork === toNetwork`).

### Private transfer specifics

- Single editable field (no second `swap-field`); the label flips with
  `direction`: `Recipient gets` (reverse, default) Ōåö `You send` (direct).
- The rate row shows the upstream-quoted counterpart with a countdown
  timer reading `validUntil`. The flip button beside the timer toggles
  `direction` so the user can type either side.
- Floating-label inputs for address + memo (`<input placeholder=" ">` +
  absolutely-positioned `<span>` driven by `:placeholder-shown`/`:focus`).
- Per-chain client-side validation via `Currency.addressRegex` /
  `extraIdRegex` (forwarded from the upstream catalog). Invalid input
  flips `data-invalid` on the field and renders an inline error.
- Action row below the input:
  - **Connected wallet** ŌĆö `window.ethereum.request({ method:
    'eth_requestAccounts' })`, gated on `useSyncExternalStore` feature
    detect. Shown for any chain (the address regex catches mismatches).
  - **QR scan** ŌĆö `getUserMedia({ facingMode: 'environment' })` opens a
    full-screen modal with the rear camera; `BarcodeDetector` decodes
    each animation frame; on hit the URI prefix is stripped
    (`bitcoin:` / `ethereum:` / `tron:` / ŌĆ”) and the bare address fills.
    Falls back to a hidden file picker when the API is missing or
    permission is denied.
  - **Paste** ŌĆö clipboard API with permission-graceful fallback.
- **Send creates a real transaction.** On click, the CTA POSTs to
  `/api/transactions` with `flow: 'fixed-rate'`, the cached `rateId`,
  the user's typed side (`fromAmount` or `toAmount` per `direction`),
  the address, and the optional extraId. On success: redirect to
  `https://changenow.io/exchange/txs/{id}` (legacy host). Submission
  errors render below the CTA; the disabled state is wired via
  `data-disabled` so the underlying `<a>` keeps the deep-link as a
  fallback for non-private modes.

### bfcache safety

The inline script in `app/[lang]/layout.tsx` listens for `pageshow` and
calls `location.reload()` when `event.persisted` is true. React effects
don't fire on bfcache restores, leaving `useHashSync`'s mount listener,
the estimate-debounce timer, the picker's outside-click handler, and
the QR scanner's video stream all attached but unresponsive. Reloading
rebuilds every effect so hitting Back from an external link feels like
a fresh visit.

## Server APIs

### `GET /api/estimate`

Proxy for `vip-api.changenow.io/v1.3/exchange/estimate`. Centralises:
- Tight input validation (`TICKER_RE`, amount cap, allowlisted promo
  codes / link IDs / `useRateId`).
- Per-IP token-bucket rate limiting (30 req / 10 s, in-process; layer a
  shared limiter in front for multi-instance deployments).
- In-flight dedupe by request signature so a burst of identical
  estimates collapses to one upstream call.
- 5 s upstream timeout.
- 15 s edge cache (`s-maxage=15, stale-while-revalidate=45`) for 200s.
- Forwards `source=private-transfers` when the request is same-asset
  same-network so the upstream quotes a real fixed-rate fee.
- Computes `cashbackUsd` from a parallel `/v1/cashback/estimate` call
  multiplied by the cached NOWŌåÆUSD price (`getNowUsdPrice`,
  `unstable_cache`-backed for 5 min).

### `POST /api/transactions`

Proxy for `vip-api.changenow.io/v1.1/transactions`. Hard-pins
`provider: 'default'` and `source: 'private-transfers'` server-side so
the browser can't escalate the attribution. Validates:
- Tickers and networks against `TICKER_RE`.
- Address against `ADDRESS_RE` (broad allowlist that covers every
  chain's canonical format but blocks shell metacharacters).
- ExtraId against `EXTRA_ID_RE`.
- Amounts with `MAX_AMOUNT` ceiling.
- `rateId` against `RATE_ID_RE` (mandatory for `flow: 'fixed-rate'`).

Forwards `flow`, `type`, and `rateId` so the upstream can settle the
counterpart at the quoted rate. 8 s upstream timeout. Returns a slim
shape: `{ id, payinAddress, payoutAddress, fromAmount, toAmount,
validUntil }` ŌĆö exactly what the client needs to redirect or render an
inline next-step splash. Upstream 4xx errors propagate with the
original status; transport failures map to 502.

### `POST /api/warm-cache`, `POST /api/tick`, `POST /api/revalidate-currencies`

Internal cache management. `warm-cache` revalidates a tag-scoped Strapi
fetch on demand, `tick` is the cron entry, `revalidate-currencies`
invalidates the `unstable_cache` entry for the currencies catalog.

## Local development

Prerequisite: an `.npmrc` with the `@changenow:` registry token. Copy `.npmrc.example` and add your GitLab token, or copy from a sibling project (the file is gitignored).

```bash
npm install
npm run dev
```

`predev` runs first and copies `cn-ui-kit` assets into `public/`, then `next dev` boots Turbopack.

App at <http://localhost:3000>. Default locale is English (no prefix); other locales live at `/{ru,fr,es,ŌĆ”}/`.

## Running in Docker

The image is a 3-stage standalone build mirroring `changenow-blog`'s. It needs the GitLab registry credentials at build time only ŌĆö they're passed in as build args and the resulting `.npmrc` is deleted before the layer is finalized.

### Build

```bash
docker build \
  --build-arg CI_SERVER_HOST=gitlab.qetuoadgjl.com \
  --build-arg CI_JOB_TOKEN=glpat-XXXX \
  --build-arg CONTENT_API_BASEURL=https://content-api.changenow.io \
  -t changenow-2-0:local \
  .
```

`CI_SERVER_HOST` / `CI_JOB_TOKEN` are the same vars Jenkins exposes ŌĆö locally, drop in any token with read access to the `@changenow/*` packages.

### Run

```bash
docker run --rm -p 3000:3000 \
  -e SITE_URL=https://changenow.io \
  changenow-2-0:local
```

The runner stage runs as a non-root `nextjs` user (uid 1001) and exposes port 3000 via Next's `output: 'standalone'` `server.js`.

### Compose (optional)

Minimal `docker-compose.yml` for local prod-like runs:

```yaml
services:
  web:

        CI_SERVER_HOST: gitlab.qetuoadgjl.com
```

## CI/CD

`deploy/build.groovy` is a Jenkinsfile that fans out to two downstream jobs:

- **staging** ŌĆö every branch push (`env.TAG_NAME == null`)
- **production** ŌĆö git tag pushes, plus `main` / `master`

Both jobs notify `#cn-front-deploy` on Slack with success/failure status.

## Notes & gotchas

- **Route segment configs must be literals.** `export const revalidate = 60` works; `export const revalidate = CACHE_SHORT` does not ŌĆö Next 16 statically analyzes these and fails the build with a vague *"Invalid segment configuration"* error otherwise.
- **Turbopack is the default in Next 16** for both `dev` and `build`. `next.config.ts#webpack` is silently ignored ŌĆö that's why `copy-webpack-plugin` from the blog is replaced by `scripts/copy-kit-assets.mjs` wired into `prebuild`.
- **i18n stubs.** `lib/server/i18n/<locale>.json` ship as empty `{}` and get filled at build time by `npm run build:i18n`. If the upstream API is unreachable, the build proceeds with whatever's already on disk (worst case: keys render as defaults / empty strings).
- **Footer skinning.** The kit ships its own design tokens; we override only colour-bearing rules in `components/layout/footer-overrides.css` so the rest of the kit's layout/typography keeps working.
